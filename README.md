## Why it's built this way

The portfolio is the public business card of an infrastructure engineer. The design constraints follow from that:

- **Defensible in interviews.** Every claim, project framing and metric must be one the owner can stand behind without overpromising.
- **Reads as enterprise IT, not a frontend showcase.** Sober dark theme, warm orange accent (`#fb923c`), no cyberpunk / gamer / matrix-rain tropes.
- **GitHub-Pages-ready out of the box.** Zero build pipeline. The repo IS the deploy artefact.
- **No third-party tracking, no analytics SDK, no ad pixels.** Privacy by absence.
- **Performance budget tiny.** All custom assets together are well under 30 KB transferred.

## Stack

| Layer | Choice | Why |
|---|---|---|
| Markup | Hand-written HTML5 | Single source of truth, lints in any editor, no build to break. |
| Styling | Vanilla CSS, CSS variables | One token swap (`--accent`) reskins the whole site. No SASS / PostCSS / Tailwind needed. |
| Interactivity | Vanilla JS, no framework | Tiny, no hydration cost, runs everywhere. |
| Background | Inline SVG generated by JS | Cheap, GPU-friendly, themable through CSS variables. |
| Hosting | GitHub Pages (master branch) | Free, HTTPS by default, low ops overhead. |
| Fonts | Google Fonts (DM Sans + JetBrains Mono) | Two weights preloaded, system fallbacks otherwise. |

Total third-party JavaScript executing on the site: **zero**.

## Features

### Animated network background

`assets/bg-network.{css,js}` — a deliberately understated visual layer that suggests "enterprise infrastructure" without bleeding into the content.

- **Three depth tiers** (`--far`, `--mid`, `--near`) drawn as separate SVG layers, each with its own opacity profile and parallax magnitude.
- **Mouse-driven parallax** on devices with hover + fine pointer; rAF-driven, throttled, paused when the tab is hidden.
- **Traveling light packets** ride along selected edges via SVG `animateMotion` — suggests live traffic between nodes.
- **Whole-background breathing** via a slow CSS opacity oscillation (~22 s).
- **Two abstract clusters** on the near layer evoke mini-topologies / rack stacks.
- **Respects `prefers-reduced-motion`** — disables animations and parallax, keeps the static topology.
- **Mobile (≤ 640 px)** drops the far layer to spare the GPU.
- **Inert to interaction** (`pointer-events: none`, `aria-hidden`).
- **Inherits `var(--accent)`** so a single colour-token change reskins it.

### Hero role rotator (typewriter)

`assets/script.js#rotateHeroRoles` + `assets/premium.css`.

- Reads the active roles list from `window.__portfolioRoles` (populated by `i18n.js`).
- 7 roles cycled — French and English titles intermixed in FR mode, fully English in EN mode.
- Typewriter behaviour: 45 ms per char, 25 ms delete, 1400 ms hold full, 280 ms hold empty.
- Blinking accent cursor via CSS `steps(2)` animation.
- Stable min-height — no layout shift between roles.
- Under `prefers-reduced-motion`, shows a static composite label and skips the loop.
- Listens to `i18n:lang-changed` so language switches mid-cycle pick up the new roles list on the next role.

### Bilingual FR / EN

`assets/i18n.js` is a 600-line vanilla module that gives the whole portfolio a French ↔ English toggle, with zero dependencies.

- **Dictionary-based.** A single `T = { fr, en }` object holds ~180 keys covering nav, hero, sections, projects, footer, form labels, placeholders, aria labels, meta tags and the rotator fallback.
- **Markup conventions** (each attribute targets one DOM property):
  - `data-i18n="key"` → `textContent`
  - `data-i18n-html="key"` → `innerHTML` (used where the source has `<strong>`)
  - `data-i18n-aria="key"` → `aria-label`
  - `data-i18n-title="key"` → `title`
  - `data-i18n-placeholder="key"` → `placeholder`
  - `data-i18n-content="key"` → `content` (meta tags)
  - `data-i18n-alt="key"` → `alt`
- **Default language is French.** The HTML literally contains French text, so the site works perfectly with JavaScript disabled — French is served as the no-JS fallback.
- **Detection order:** `localStorage` → `navigator.language` → `fr`.
- **Persistence:** the choice is stored under `lm-portfolio-lang`. Survives reloads and cross-page navigation.
- **Toggle UI:** small `FR | EN` pill in the nav, just before the CV CTA. On mobile it lives inside the hamburger menu via the existing `order: -1` rule.
- **Live event:** every `applyLang()` call dispatches `i18n:lang-changed` on `window` — the hero rotator listens to this to refresh its roles list mid-cycle.
- **Dynamic `<html lang>`** — updated on every switch for screen readers and search engines.

### UI polish

`assets/premium.css` layers additive microinteractions on top of `style.css`:

- Hover lift (`translateY(-2px)`) + soft accent shadow on featured / project / expertise / info cards.
- AI cards get a thin orange sweep line that slides in on hover.
- Focus-visible outlines in `--accent` on every interactive control.
- All transforms disabled under `prefers-reduced-motion`.

## SEO & i18n strategy

GitHub Pages doesn't give us server-side language routing, so the strategy is deliberately pragmatic:

- **Canonical URL** stays the same per page; the language is a client-side toggle on top.
- **`<link rel="alternate" hreflang>` tags** declare the FR and EN variants of each page (with `?lang=en` as the EN URL hint). The `x-default` points to the FR canonical.
- **Per-language meta tags** — `<title>`, `<meta name="description">`, `og:title`, `og:description`, `twitter:title`, `twitter:description` all carry `data-i18n*` attributes and swap when the user toggles.
- **`<meta property="og:locale">`** + `og:locale:alternate` declare both locales for social previews.
- **Keyword targeting** kept in French for the primary recruiter audience (SCCM, Intune, alternance, ESIEE-IT…), with English fallbacks for international readers.
- **No analytics, no tracking scripts** — Google Search Console and Bing Webmaster work fine without them via the sitemap/robots conventions of GitHub Pages.

> **Caveat.** Because language is client-side only, search-engine crawlers without JavaScript execution will index the French version. For full bilingual SEO, you'd need separate `/en/` paths — see [Known limitations & roadmap](#known-limitations--roadmap).

## Security & privacy

- **No tracking.** No Google Analytics, no Plausible, no Tag Manager, no Facebook pixel. Zero outbound network calls beyond fonts.googleapis.com and fonts.gstatic.com (Google Fonts).
- **No cookies.** Only `localStorage` is used, and only to remember the language preference (key: `lm-portfolio-lang`).
- **No form submission to a third-party endpoint.** The contact form (`contact.html`) is currently presentational. Any future submission target should use a privacy-respecting service (Formspree, plain `mailto:`, or a self-hosted endpoint).
- **No inline `eval`, no `dangerouslySet*`-equivalents.** `data-i18n-html` writes to `innerHTML` from a dictionary that is part of the source code — there is no user-controlled input that flows into it. A reader-only Content Security Policy is therefore easy to add when needed.
- **No third-party JavaScript.** Everything that executes is in this repo and reviewable in one sitting.
- **Subresource integrity** is not used for the Google Fonts stylesheet because Google rotates the underlying CSS file. If you self-host the fonts, add SRI hashes.
- **`rel="noopener"`** is applied to every external link that opens in a new tab — prevents the opened page from getting access to `window.opener`.
- **HTTPS only** when served from GitHub Pages.
- **NDA badge** on project cards covered by SYSTRA's confidentiality clause — those projects display a visible "Private code · NDA" indicator instead of a fake GitHub link.

### Suggested CSP (when you control headers)

```
default-src 'self';
style-src 'self' 'unsafe-inline' https://fonts.googleapis.com;
font-src https://fonts.gstatic.com;
script-src 'self';
img-src 'self' data:;
connect-src 'self';
frame-ancestors 'none';
base-uri 'self';
form-action 'self';
```

GitHub Pages does not let you set custom HTTP headers, so this CSP would require a CDN in front (Cloudflare, Vercel, Netlify…).

## Accessibility

- Semantic landmarks: `<nav>`, `<main>` via `<section>` blocks, `<footer>`, `<button>` for actions, `<a>` for navigation.
- `aria-label` on icon-only buttons (social icons, hamburger, scroll-top, language toggle).
- `aria-pressed` on the language toggle buttons reflects active state.
- `aria-live="polite"` on the rotating role span so assistive tech announces role changes without interrupting.
- `aria-hidden="true"` on purely decorative elements (background SVG, role-stack mini-line, scroll progress bar, animated bg).
- All animations honour `prefers-reduced-motion: reduce`.
- Focus ring (`outline: 2px solid var(--accent)`) on every interactive control via `:focus-visible`.
- Tap targets ≥ 36×36 px on mobile.
- Contrast checked: `--text` (#ebedf2) on `--bg` (#0b0d12) clears WCAG AAA for body text.

## Performance

Custom asset budget when fully loaded:

| File | Size (uncompressed) | Role |
|---|---:|---|
| `index.html` | ~38 KB | Markup |
| `projects.html` | ~32 KB | Markup |
| `contact.html` | ~9 KB | Markup |
| `style.css` | ~50 KB | Base design system |
| `premium.css` | ~10 KB | Iteration polish + lang toggle + rotator |
| `bg-network.css` | ~5 KB | Background animations |
| `bg-network.js` | ~7 KB | Background renderer + parallax |
| `i18n.js` | ~25 KB | FR + EN dictionary + engine |
| `script.js` | ~9 KB | Scroll/reveal/rotator |

Total custom payload ≈ **120 KB uncompressed**, **~35 KB over the wire after gzip**.

Notes:
- `bg-network.js` and `i18n.js` are `defer`-loaded — they don't block paint.
- `bg-network.js` is paused when `document.hidden` becomes true (mouse parallax loop short-circuits).
- The hero avatar uses `fetchpriority="high"` for early LCP.
- Reveal-on-scroll animations use IntersectionObserver, not `scroll` listeners.
- Scroll progress bar transforms `scaleX` (GPU-composited) — no layout thrash.

## Local development

You need any static HTTP server. Python 3 ships with one:

```bash
python -m http.server 8765
# then open http://localhost:8765/index.html
```

Or with Node:

```bash
npx serve .
```

Any change to a file is reflected on next reload — no watchers, no recompile.

### Browser testing checklist

1. Toggle FR ↔ EN — every visible string should swap, including page title and OG meta tags.
2. Refresh after switching — language must persist (localStorage check).
3. Open in a different browser / private window — `navigator.language` should drive the first paint.
4. Resize to 390 px — toggle must remain reachable (it lands inside the hamburger menu on mobile).
5. Enable "Reduce motion" in OS / DevTools → Rendering → Emulate CSS — bg packets and parallax must stop, hero rotator must show the static fallback label.
6. Open DevTools Console — should be empty except a 404 on `/favicon.ico` (acknowledged, pre-existing).

## Customisation guide

### Change the accent colour

Edit `assets/style.css`:

```css
:root {
  --accent: #fb923c;          /* current warm orange */
  --accent-strong: #fdba74;
  --accent-glow: rgba(251, 146, 60, 0.12);
  --accent-glow-strong: rgba(251, 146, 60, 0.22);
}
```

That single block propagates to nav, buttons, focus rings, links, hero badge, scroll progress bar **and the network background**.

### Add a new project to `projects.html`

1. Copy an existing `<div class="project-card">` block.
2. Set `data-category` to one of: `ai`, `infra`, `reseaux`, `securite`, `scripts`, `cloud`. Filter buttons pick this up automatically.
3. Replace title, description, tags, and the `contact.html?project=...` slug for prefilling the contact form.
4. To translate it, add two new keys (`pjs.xxx.title`, `pjs.xxx.desc`) to **both** `T.fr` and `T.en` in `assets/i18n.js`, then set `data-i18n="pjs.xxx.title"` on the project title node.

### Change a hero role in the rotator

Edit the `ROLES` constant in `assets/i18n.js` — add to both `ROLES.fr` and `ROLES.en` so the typewriter never references a missing entry.

### Update the reduced-motion fallback line

Edit `rotator.fallback` in both `T.fr` and `T.en`. It's what assistive-tech users and motion-sensitive users see in place of the typewriter.

### Wire the contact form to a real endpoint

Currently `contact.html`'s `<form id="contact-form">` has no `action` — it's presentational. Plug in:

- **Formspree** — set `action="https://formspree.io/f/YOUR_ID"` and `method="POST"`.
- **mailto** — set `action="mailto:manivit.lucas@gmail.com"` and `method="POST"` (works but UX is browser-dependent).
- **Custom backend** — point to your endpoint and add CSRF if needed.

## Branch & release workflow

The repo follows a feature-branch flow into `master`. `master` is the live GitHub Pages branch.

```
master                                  # production, served by GitHub Pages
├── feature/animated-tech-background    # initial subtle bg layer (merged via PR #1)
├── feature/premium-ai-enterprise-portfolio  # bg V2, orange palette, AI sections, hero rotator (merged via PR #1)
└── feature/i18n-fr-en-toggle           # this branch — bilingual support
```

To ship a change:

```bash
git checkout -b feature/your-thing
# edit files
git add -p
git commit
git push -u origin feature/your-thing
gh pr create -B master                  # or open in the GitHub UI
# review, then merge — GitHub Pages redeploys within a minute
```

Hard rule: **never push directly to `master`**. The merge commit on master is the deploy.

## Known limitations & roadmap

| Item | Current state | Future option |
|---|---|---|
| EN SEO | Client-side toggle. Crawlers without JS index FR only. | Generate static `/en/` paths via a build step or GitHub Action; emit per-language `<link rel="canonical">`. |
| Contact form | Presentational, no backend. | Wire to Formspree / Resend / self-hosted endpoint. |
| Font self-hosting | Google Fonts CDN. | Self-host with `@font-face` for full privacy + SRI. |
| Analytics | None. | Add Plausible or Umami when needed — both privacy-friendly, cookieless. |
| Sitemap & robots.txt | Not present. | Generate `sitemap.xml` + `robots.txt` for cleaner indexing. |
| 404 page | None. | Add `404.html` — GitHub Pages serves it for missing paths. |
| CV PDF | Single FR version in repo root. | Provide an English version (`CV-Lucas-Manivit-EN.pdf`) and switch the link based on `window.__portfolioLang`. |
| Light theme | Not built — dark only by design. | Could add via a second `:root[data-theme="light"]` block if ever needed. |
